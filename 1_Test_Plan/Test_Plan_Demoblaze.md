## 1. Introduction
This document describes the test plan for the Demoblaze demo e-commerce website. 
The purpose of this test plan is to define the testing strategy, scope, resources, and schedule for manual testing activities.

## 2. Objectives
The main objectives of testing are:
- To verify that core e-commerce functionalities work as expected.
- To identify defects in critical user flows.
- To ensure a good user experience for end users.

## 3. Scope

### In Scope
- User registration
- User login and logout
- Product categories and product listing
- Product details page
- Add to cart functionality
- Cart management

### Out of Scope
- Payment gateway integration (real payments)
- Performance testing
- Security testing
- Mobile application testing

## 4. Test Items
The following modules will be tested:
- Home Page
- Login Module
- Signup Module
- Product Page
- Cart Page
- Checkout / Place Order

## 5. Test Approach
Manual black-box testing will be performed based on functional requirements.
Test cases will be designed using:
- Equivalence Partitioning
- Boundary Value Analysis
- Positive and negative scenarios

Testing types:
- Functional testing
- UI testing
- Smoke testing
- Regression testing

## 6. Test Environment
- Browser: Chrome, Firefox
- OS: Windows 10 / 11
- Test URL: https://demoblaze.com
- Test Type: Manual testing

## 7. Entry and Exit Criteria

### Entry Criteria
- Test environment is available.
- Test cases are prepared and reviewed.
- Application is accessible.

### Exit Criteria
- All planned test cases are executed.
- All critical defects are fixed.
- Test summary report is prepared.

## 8. Deliverables
- Test Plan
- Test Cases document
- Bug reports
- RTM (Requirement Traceability Matrix)

## 9. Roles and Responsibilities
- QA Engineer: Prepare test plan, design test cases, execute tests, report bugs.
- Developer: Fix reported defects.
- Project Manager: Review test progress.

## 10. Risks and Mitigation
| Risk | Mitigation |
|------|------------|
| Application changes frequently | Update test cases regularly |
| Limited documentation | Perform exploratory testing |

## 11. Assumptions
- Demoblaze is a stable demo application.
- No real financial transactions will be performed.
